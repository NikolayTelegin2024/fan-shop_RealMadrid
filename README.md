2. Тематика сайта, название

Тематика: учебный веб‑сервис — прототип фан‑шопа ФК «Реал Мадрид», демонстрирующий навыки серверной разработки на Flask. Может использоваться фанатами клуба для ознакомления с ассортиментом атрибутики.

Название сайта: «Реал Мадрид Россия: Официальный фан‑шоп» (учебный проект).

3. Цель и задачи системы

Цель: разработка веб‑сервиса с использованием Flask и сопутствующих технологий для получения практических навыков серверной разработки в рамках единого проекта. Проект также может служить демонстрационным ресурсом для фанатов клуба.

Задачи:

Реализовать серверную часть на Flask с шаблонами Jinja2.
Настроить работу со статическими файлами (CSS, JS, изображения атрибутики клуба).

Разработать формы для взаимодействия пользователей (регистрация, авторизация, корзина).

Создать базу данных для хранения информации о товарах, пользователях, заказах.

Реализовать систему аутентификации пользователей.

Разработать REST API для взаимодействия фронтенда и бэкенда.

Обеспечить адаптивный интерфейс с демонстрационным функционалом (без реальной оплаты).

Создать контент, интересный для фанатов (новости клуба, фотогалереи, статистика).

5. Содержание сайта и структура БД (Пока все примерно, в процессе реализации возможны измененения)

Структура базы данных (SQLite):

Users (id, username, email, password_hash, registration_date, bonus_points);

Products (id, name, description, price, category, image_url, in_stock, season);

Orders (id, user_id, status, total_price, created_at);

OrderItems (id, order_id, product_id, quantity);

Cart (id, user_id, product_id, quantity);

News (id, title, content, published_date, image_url).

Основные разделы сайта:


1)Главная страница (/):

анимированный слайдер с акциями и новинками;

блок «Хиты продаж» (товары с наибольшим количеством добавлений в корзину);

анонсы матчей и новостей клуба.

2)Каталог товаров (/catalog):

динамическая генерация карточек товаров из БД;

фильтры по категориям, сезону, размеру, цене;

сортировка (по цене, популярности, новизне).

3)Карточка товара (/product/<id>):

фотогалерея (3–5 изображений);

описание, цена, наличие, сезон;

кнопка «В корзину» (отправляет POST‑запрос к REST API).

4)Корзина (/cart):

список выбранных товаров (данные из БД);

изменение количества (AJAX‑запрос к API);

очистка корзины (POST‑запрос);

расчёт итоговой стоимости (виртуально).

5)Авторизация (/login, /register):

формы с валидацией (Jinja2);

хеширование паролей (Flask‑Login);

сессия пользователя.

6)Личный кабинет (/profile):

история заказов (фиктивные данные из БД);

бонусные баллы (виртуальный счётчик);

настройки профиля (смена пароля, email).

7)Новости клуба (/news):

статьи о матчах, трансферах, событиях клуба (на русском);

фотогалерея с матчей.

8)Админ‑панель (/admin):

просмотр пользователей, товаров, заказов, новостей;

добавление/редактирование товаров и новостей (формы Flask‑WTF);

статистика (количество пользователей, просмотров, добавлений в корзину).


6. Функциональные возможности и технологии (Пока все примерно, в процессе реализации возможны измененения)

1)Серверная часть (Flask):

Маршрутизация: Flask routes для всех разделов.

Шаблоны: Jinja2 для динамического контента (наследование шаблонов, блоки).

Статические файлы: CSS (стилизация), JS (AJAX), изображения (атрибутика клуба).

Формы: Flask‑WTF для регистрации, авторизации, редактирования товаров и новостей.

Аутентификация: Flask‑Login для сессий, хеширование паролей.

База данных: SQLite + SQLAlchemy ORM.

REST API: endpoints для:

    /api/products — получение списка товаров;

    /api/cart — управление корзиной (GET/POST/DELETE);

    /api/orders — создание заказа (демо);

    /api/news — получение новостей клуба.

2)Клиентская часть:

AJAX‑запросы к REST API (добавление в корзину без перезагрузки страницы).

Адаптивный дизайн (Flexbox/Grid).

Анимации: добавление товара в корзину (JS + CSS).

Валидация форм на стороне клиента и сервера.

Интерактивные элементы: фильтры с мгновенным обновлением каталога, подсветка активных фильтров.
